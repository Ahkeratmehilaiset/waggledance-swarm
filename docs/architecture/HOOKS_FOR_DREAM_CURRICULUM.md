# Hooks for Dream Curriculum (Session C)

```
contract_version: 1
```

## Revision policy

- **Additive changes** bump the minor version (1.x).
- **Breaking changes** require a new file `HOOKS_FOR_DREAM_CURRICULUM_v2.md` plus migration notes. Consumers MUST opt in by declaring `version: 2` in their session state.

## Consumer declaration enforcement

Future sessions (Session C and beyond) that read self-model outputs **must** declare in their own `state.json`:

```json
"consumed_hook_contracts": [
  {
    "file": "docs/architecture/HOOKS_FOR_DREAM_CURRICULUM.md",
    "version": 1
  }
]
```

If a consuming session sees a `contract_version` it has not declared:
- **fail loud**, do not silently adapt
- record the mismatch in `state.json` and abort

## Session C dependency surface

The dream curriculum reads exactly these fields from the self-model snapshot. Adding new fields here is additive (1.x); changing a type or removing a field is breaking (v2).

### `attention_focus[]`

```
[
  {
    "curiosity_id": str,
    "candidate_cell": str | null,
    "estimated_value": float,
    "suspected_gap_type": str,
    "count": int
  }
]
```

`suspected_gap_type` is one of: `missing_solver`, `improvement_opportunity`, `bridge_composition`, `unit_family_mismatch`, `contradiction_surface`, `low_confidence_routing`, `subdivision_pressure`, `meta_solver_opportunity`.

### `workspace_tensions[]`

```
[
  {
    "tension_id": str,                 # 12 hex; structural identity
    "type": str,
    "claim": str,
    "observation": str,
    "severity": "low" | "medium" | "high",
    "resolution_path": str,            # see enum below
    "lifecycle_status": str,           # see enum below
    "evidence_refs": [str, ...]
  }
]
```

`resolution_path`: `calibration_correction` | `blind_spot_promotion` | `deferred_to_dream` | `requires_human_review`.
`lifecycle_status`: `new` | `persisting` | `resolved`.

**Critical Session C rule:** tensions with `resolution_path = "deferred_to_dream"` are NOT marked resolved by Session C generation alone. They are resolved only when a later snapshot sees artifact-observable reality change such that the tension no longer exists. This preserves the deterministic boundary between offline dream generation and observed self-model change.

### `blind_spots[]`

```
[
  {
    "domain": str,                     # matches expected_domains.yaml domain_id
    "severity": "low" | "medium" | "high",
    "detectors": [str, ...],           # subset of {coverage_negative_space, curiosity_silence}
    "description": str,
    "provenance": {
      "detectors": [str, ...],
      "has_structural_evidence": bool,
      "expected_capability_signals": [str, ...]
    }
  }
]
```

### `calibration_evidence` per scorecard dimension

```
{
  "dimension": str,
  "evidence_implied_score": float | null,
  "evidence_refs": [str, ...],
  "calibration_status": "ok" | "unavailable" | "mismatch",
  "notes": str
}
```

Session C should treat `calibration_status="unavailable"` as a hard skip (no dream curriculum on undermined dimensions).

### `self_model_delta`

```
{
  "delta_status": "bootstrap" | "identical_inputs_to_previous" | "history_corrupted" | "computed",
  "snapshot_id": str,
  "previous_snapshot_id": str | null,
  "scorecard_movements": [
    {"dimension": str, "previous": float, "current": float, "delta": float}
  ],
  "new_blind_spots": [str, ...],
  "resolved_blind_spots": [str, ...],
  "new_tensions": [str, ...],
  "resolved_tensions": [str, ...]
}
```

Session C MUST refuse to generate dream curriculum when `delta_status == "history_corrupted"`.

## Forward Dream Pipeline Constraints

1. **Stochastic generation phase remains offline.** No live runtime call. No port 8002.
2. **Deterministic collapse goes through gated validation.** Any candidate proposal generated by Session C must pass `tools/propose_solver.py` before any further consideration.
3. **Counterfactual replay uses pinned historical artifacts.** Read snapshots through their `pin_hash`. Never re-glob.
4. **Meta-proposal artifacts remain machine-readable and auditable.** Session C MUST emit JSONL or sorted JSON; never opaque blobs.
5. **Runtime flip remains out of scope and separately gated.** Dream curriculum cannot move data into the runtime read path. That requires a separate reviewed commit after the campaign and after explicit human approval.

## What Session C MUST NOT do

- Mark `deferred_to_dream` tensions resolved without artifact-observable change.
- Mutate `HISTORY.jsonl`. The history is append-only and signed.
- Bypass `tools/propose_solver.py`.
- Touch port 8002 or any runtime adapter.
- Assume Ollama or a live FAISS index is available in tests.
- Run the full ~5817 test suite.

## What Session C CAN do

- Read `attention_focus`, `workspace_tensions`, `blind_spots`, `calibration_evidence`, `self_model_delta` deterministically.
- Generate counterfactual proposals against pinned snapshots.
- Append to a new `dream_curriculum.jsonl` (subject to its own append-only chain).
- Emit `dream_curriculum_proposal.json` artifacts that reference back into a self-model `pin_hash` via `continuity_anchor`.
- Gate every emission through the proposal quality gate.

## Versioned schema notice

This contract is `1`. The Session B implementation in this branch produces snapshots that match this contract exactly. Any divergence found by a Session C consumer is a bug here, not in Session C.
